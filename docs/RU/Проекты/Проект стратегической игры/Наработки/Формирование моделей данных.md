## Вступление (при ознакомлении с Issue #15 в репозитории - пропустить)
Третий принцип проекта (свобода) требует создания огромного количества классов игровых сущностей для реализации творческого потенциала игроков в полной мере. Вручную создать такое количество сущностей, а потом поддерживать их, за разумный срок возможным не представляется. Как быть?

Мне пришла идея, что каждый класс игровых сущностей может быть порождением декларативной функции, основываясь на IPO модели (упомянуто в Issue [#14](https://github.com/Eisenprogrammierer/Strategy-game-project/issues/14)), на основании правил порождающей грамматики и словаря шаблонов. При таком подходе можно получить воспроизводимую, обоснованную процедуру генерации классов игровых сущностей, где произвол разработчика сведён к минимуму, зато возможности для разработки огромны, без необходимости придумывать что-то самостоятельно или через ИИ.

Такая архитектура гарантирует комбинаторный взрыв количества возможных классов игровых сущностей, как того и требует третий принцип проекта. Например, если у одного класса игровых сущностей 8 черт характеристики, каждая из которых обладает хотя бы 4 состояниями, то количество возможных комбинаций равняется: 4^8 = 65536. Говоря в терминах игры, допустим, у нас есть класс - танки. Допустим, они характеризуются по 8 параметрам: экипаж, запас хода, вооружение, скорость, защищённость, средства связи, автоматизация, стоимость производства. Каждый из этих параметров допустим описывается 4 состояниями (спорно, но это пример, на скорую руку). Итого мы получаем до 65536 уникальных моделей танков (не экземпляров игровых объектов, а их классов).

Правила порождающей грамматики и состояния характеристик можно определять через JSON, что даёт возможность модифицировать игровые сущности без необходимости внесения изменений в движок ([#11](https://github.com/Eisenprogrammierer/Strategy-game-project/issues/11)). Однако это требует гибкого API движка.

## Некоторые архитектурные соображения по декларативной грамматике

Можно исходя из IPO-модели определить самый базовый алфавит черт класса игровой сущности, применимый к любому классу. При этом, каждый класс игровой сущности имеет уникальный набор этих черт, поэтому, можно создать специализированный алфавит, конкретизирующий IPO-модель аналогично принципу наследования в ООП. Но даже с таким подходом, остаётся открытым вопрос определения этих черт.
Можно определить набор доменов среды, с которыми взаимодействует игровая сущность. За основу можно было бы взять три домена: материя, энергия и информация. Любая игровая сущность (будучи моделью реального объекта) состоит из материи и взаимодействует с материей, энергия является мерой перехода материи из одного состояния в другое, а информация - это форма абстрактного представления конкретного объекта, являющаяся обязательным условием управляемого взаимодействия объектов (игровые сущности проектируются исходя из допущения, что они управляемы). Это число не является ограниченным. По желанию или необходимости, можно будет добавлять новые домены - например, магию для фэнтезийных сценариев, киберпространство для фантастических сценариев, что угодно. Но эти три домена - это фундамент, который присутствовать должен всегда.
Далее, исходя из этих доменов, можно дополнять IPO-модели классов сущностей. В качестве примеров возьмём государство, завод и батальон:

Домены:

| Сущность    | Материя                     | Энергия                                          | Информация                                                                       |
| ----------- | --------------------------- | ------------------------------------------------ | -------------------------------------------------------------------------------- |
| Государство | Людские и природные ресурсы | Труд людей, преобразование ресурсов в изделия    | Законы, приказы, постановления                                                   |
| Завод       | Работники и сырьё           | Труд работников, изготовление из сырья продукции | Заказы, правила внутреннего распорядка, рыночные цены на сырьё и труд работников |
| Батальон    | Солдаты и вооружение        | Боевые действия и выполнение приказов            | Приказы, инструкции, пропаганда, оперативная обстановка                          |

Исходя из этих данных складываются такие IPO-модели:


| Сущность    | Вход                                                       | Обработка                                                             | Выход                                 | Состояние                                                                                   |
| ----------- | ---------------------------------------------------------- | --------------------------------------------------------------------- | ------------------------------------- | ------------------------------------------------------------------------------------------- |
| Государство | Рождаемость, иммиграция, недра на своей территории         | Профильные министерства (образования, здравоохранения), разведка недр | Эмиграция, экспорт                    | Численность населения и ресусов                                                             |
| Завод       | Трудовые ресурсы, их количество и качество, сырьё          | Рабочие цеха, конвейеры, станки, другие средства производства         | Готовая продукция                     | Численность работников, их профессионализм, бюджет завода                                   |
| Батальон    | Приказы, оперативная обстановка, вооружение и комплектация | Выполнение приказов и боевых задач                                    | Вклад в достижение поставленных целей | Количество готовых к бою, раненых, убитых, количество рабочей и выведенной из строя техники |

При таком подходе мы получаем достаточно точную, воспроизводимую процедуру формирования моделей данных классов игровых сущностей. 

## Вопрос конкуренции игровых сущностей

Из текущей модели логически никак не следует, что сущности конкурируют между собой, что противоречит действительности: конкуренция в истории человечества в тех или иных формах была всегда. Для этого в модель можно ввести четвёртый домен - сила. Сила является производной от отношения целевой функции сущности к вызовам среды. С целью формирования целевой функции, введём дополнительную аксиому: любая сущность, обладающая способностью самовоспроизводства и самоподдержания, стремится к выживанию и расширению своего влияния. Этому всегда противоречат некоторые аспекты среды: другие аналогичные конкурирующие сущности, неблагоприятные условия, прочее.

Пусть дано множество сущностей $E$. Для каждой сущности $e_i∈E$ определён **вектор внутреннего состояния**:

$Si(t)∈Σi​$

где $Σi$ — пространство состояний

### Аксиома 1 (Целевая функция).

Для каждой сущности $e_i$ существует функция полезности:

$$U_i:Σ_i→R$$

которая численно выражает «степень удовлетворённости» сущности своим состоянием. Сущность стремится максимизировать $Ui$​. Для простоты можно считать, что $U_i$ монотонно убывает при потере ресурсов, населения, территории и монотонно возрастает при их приросте.

### Аксиома 2 (Среда и ресурсы)

Среда предоставляет конечное множество ресурсов $R(t)$ распределённых по вокселям или регионам. В каждый момент времени ресурсы конечны:

$$∑iR_i(t)≤R_{max}⁡(t)$$

Сущность $e_i$​ может извлекать ресурсы со скоростью, ограниченной её состоянием:

$$r_i(t)=f_i(S_i(t))≥0$$

### Аксиома 3 (Конфликт интересов)

Если две сущности $e_i$​ и $e_j​$ претендуют на один и тот же ресурс $R_k$​ и этот ресурс не может быть поделён без ущерба для одной из сторон (неделимость или убывающая отдача от дележа), возникает **конфликт интересов**.

Формально: сущности $e_i$​ и $e_j$​ находятся в конфликте, если существует хотя бы один ресурс $Rk$​, для которого:

$$\Delta U_i \Delta R_k⋅\Delta U_j \Delta R_k>0$$

(обе получают положительную полезность от обладания этим ресурсом). И при этом ресурс не может быть одновременно использован обеими без ущерба:

$$R_k(i)+R_k(j)≤R_{k_{total}}$$ $$R_k(i)⋅R_k(j)=0$$
(для неделимых ресурсов)


### Аксиома 4 (Домен силы)

Для разрешения конфликтов сущность может тратить часть своих ресурсов на создание **силового потенциала** $F_i(t)$.

**Силовой потенциал** — это агрегированная характеристика, выводимая из состояния:

$$F_i(t)=g_i(S_i(t))$$

где $g_i$​ — функция, специфичная для класса сущности. Для государства — это армия. Для танка — огневая мощь. Для завода — производственные мощности.

При конфликте за ресурс исход определяется соотношением силовых потенциалов. Вероятность победы сущности $i$ над сущностью $j$ :

$$P_i\rightarrow j=F_iα(F_iα+F_jα)$$

где $α≥1$ — параметр, определяющий «эффект масштаба» (при $α=1$ — линейная зависимость, при $α=2$ — квадратичная, отсылка к модели Ланчестера).

### Аксиома 5 (Обратная связь: сила потребляет ресурсы)

Силовой потенциал не бесплатен. Его поддержание требует затрат:

$$r_{{i}_{military}}(t)=h_i(F_i(t))≥0$$

где $h_i$​ — функция стоимости (обычно монотонно возрастающая).

Таким образом, возникает **дилемма «пушки vs масло»**: ресурсы, потраченные на силу, не могут быть использованы для прямого увеличения полезности.

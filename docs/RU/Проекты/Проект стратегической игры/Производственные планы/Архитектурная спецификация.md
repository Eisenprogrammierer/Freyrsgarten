## Определение границ системы
Игра включает в себя:
- Игровой мир. Тот, в свою очередь, делится на:
	- Граф сущностей, рёбра которых являются протоколами;
	- Среду, включающую внешние (относительно сущностей) факторы;
- Движок - интерпретатор конфигурации, инициализатор и контроллер игровой симуляции;
- Конфигурации - файлы, описывающие классы сущностей, протоколы и правила.

Что НЕ входит (пока что), но обязательно надо реализовать:
- Графический интерфейс;
- Мультиплеер;
- Редактор контента;
- Сохранения.

## Онтология
Система включает в себя нижеследующие фундаментальные типы:
- `Entity` - вершина графа, конечный автомат с битовой маской, накопителями напряжения и таблицей правил поведения;
- `Protocol` - ребро графа, адаптер для передачи сигналов между двумя сущностями;
- `Signal` - словарь, передаваемый через протокол;
- `Class` - разделяемый объект, содержащий разметку маски, правила поведения, условия метаморфозы;
- `Bitmask` - 64-битное целое с методами для чтения/записи семантических сегментов;
- `Voltage` - аналоговое значение напряжения на конкретном сегменте маски.

## Инварианты системы
- Аксиома изоляции такта: все сущности читают состояние из буфера $t$, пишут в буфер $t+1$.
    
- Аксиома протокола: никакие две сущности не общаются напрямую, только через протокол-адаптер.
    
- Аксиома атомарности переключения: изменение бита в маске происходит только при превышении порога напряжением, накопленным по строгим правилам.
    
- Аксиома сохранения энергии/материи: сумма ресурсов во всех сущностях + среде = `const`.
    
- Аксиома метаморфозы: новая сущность порождается только из существующей (или из среды) и наследует часть свойств по правилам.

## Интерфейсы
`Entity`:
	`get_segment(name) -> int`;
	`set_segment(name, value) -> None`;
	`get_voltage(name) -> float`;
	`add_voltage(name, delta) -> None`;
	`apply_signal(signal: Dict[str, int]) -> None`;
	`generate_signal() -> Dict[str, int]`.

`Protocol`:
	`transfer_down(parent, child) -> None`;
	`transfer_up(child, parent) -> None`;
	`transfer(sender, receiver) -> None`.

`World`:
	`step() -> None`.

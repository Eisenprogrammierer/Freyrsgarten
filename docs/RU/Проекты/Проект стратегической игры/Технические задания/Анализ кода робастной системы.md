
Первая робастная система движка (прототип на Python) состоит из двух подсистем:
1. Объектно-ориентированное ядро: описывает сами сущности (вершины графа), связи между ними и игровую симуляцию;
2. Функциональный пакет: калькуляторы непрерывных переменных.

## Объектно-ориентированное ядро (пакет /src/domains)

### Модуль /src/domains/entity.py

```python
Signal = Dict[str, int]

Bitmask = int

Voltage = float
```

Базовые переменные: сигнал в виде выходного алфавита некой сущности (или среды), битовая маска и напряжение.

```python
class Entity
```

Базовый класс игровой сущности: конечный автомат данных с битовой маской и напряжениями. Хранит и меняет собственное состояние, вырабатывает сигналы.

```python
def __init__(self, entity_id: str, entity_class: str, config: dict)
```
Интерфейс инициализации сущности: уникальный идентификатор, название класса сущностей (в разработке) и правила порождающей грамматики. Возвращает экземпляр класса сущностей (вот этот танк, вот этот завод) с состоянием и логикой работы.

Протокол сущности включает в себя методы:
- `get_segment` - получить битовую подмаску;
- `set_segment` - изменить битовую подмаску;
- `get_voltage` - получить значение текущего напряжения;
- `add_voltage` - увеличить напряжение;
- `_apply_schmitt_trigger` - переключить значение бита при преодолении верхнего или нижнего порога Шмитта;
- `apply_signal` - подать сигнал на контур обратной связи;
- `generate_signal` - подать сигнал на контур прямой связи;
- `apply_environment` - оказать воздействие со стороны среды;
- `evaluate_step` - обработка правил поведения (определяются порождающей грамматикой в конфигах);
- `_check_condition` - валидация состояния;
- `_execute_action` - произведение действия.

### Модуль /src/domain/protocol.py

```python
class Protocol
```

Реализация двунаправленного адаптера как ребра в графе связей сущностей. Преобразует сигналы сущностей согласно правилам порождающей грамматики.

```python
def __init__(self, protocol_id: str, config: dict)
```
Интерфейс инициализации адаптера: уникальный идентификатор и правила порождающей грамматики.

Протокол адаптера включает в себя нижеследующие методы:
- `_transform` - непосредственное преобразование сигнала;
- `compute_signal` - вычисление преобразованного сигнала;
- `transfer` - общий метод передачи;
- `transfer_down` - передача управляющего воздействия от высшей сущности на низшую (прямая связь);
- `transfer_up` - передача управляющего воздействия от низшей сущности на высшую (обратная связь)

### Модуль /src/domain/world.py

```python
class EntityClass
```

Приспособленец конфигурации классов сущностей с интерфейсом, аналогичным `Entity`.

```python
class World
```

Контроллер симуляции с фазовой буферизацией тактов.

```python
def __init__(self, environment: Dict[str, float] = None)
```

Интерфейс инициализации контроллера. Окружение задаётся правилами порождающей грамматики.

Протокол контроллера реализует нижеследующие методы:
- `register_class` - регистрация класса сущностей;
- `create_entity` - регистрация конкретной сущности на основании класса;
- `add_connection` - регистрация протокола(-ов) между сущностями;
- `_accumulate` - накопление сигналов
- `step` - один дискретный шаг симуляции (концепция ODOO)

Настоящий производственный план направлен на разработку стратегической игры (название пока не придумал) и охватывает вопросы реализации программного, информационного и математического обеспечения. Целями данного плана являются предоставление фиксированных во времени ориентиров разработки и направление работ.

## Содержание

### Фаза 1: теоретическое исследование и концептуализация (1-2 месяца)
**Цель**: создать прочную теоретическую базу, исключающую переписывание вычислительного ядра в будущем.
**Содержание работ**:
1. Изучение существующих математических моделей, релевантных целям разработки, их анализ на предмет соответствия специфике;
2. Постановка экспериментов с привлечением формальных методов и машинного обучения, адаптация моделей по необходимости;
3. Подготовка документации по моделям;
4. Разработка модуля игровых вычислений;
5. Моделирование неопределённости и тумана войны, реализация метода быстрого расчёта прогнозной траектории, проектирование механизма корректировки прогнозов;
6. Реализация первых робастных алгоритмов и их формальное обоснование.
**Результат:** документ с перечислением и описанием моделей, формальное обоснование, простой робастный алгоритм на Python или Julia.

### Фаза 2: архитектурное проектирование (14-28 дней)
**Цель**: проектирование движка с использованием методик системного анализа и математики.
**Содержание работ**:
1. Выбор технологий и отведение каждой из них конкретной роли в общей структуре;
2. Определение структур данных для игровых сущностей, процедур их сериализации;
3. Проектирование модульной архитектуры;
4. Проектирование воксельного мира, системы чанков, системы контроля и изменения территорий;
5. Проектирование API - конечных точек и протоколов обращения к ним;
6. Проектирование походового течения времени, разделения слоёв стратегической абстракции, логики ленивых вычислений;
7. Проектирование графического редактора контента.
**Результат**: технический проект (диаграммы, описание API. схема структур данных и БД, документация на компоненты), репозиторий с пустой структурой проекта.

### Фаза 3: Прототипирование (2-4 месяца)
**Цель**: получить минимально играбельную версию.
**Содержание работ**:
1. Реализация движка, основываясь на результатах работы фаз 1 и 2;
2. Реализация минимального игрового интерфейса и карты;
3. Базовый геймплейный цикл, простейшие доклады, первые примитивные персонажи;
4. Разделение уровней управления, реализация интерфейсов для них.
**Результат**: доказательство работы механик игры, первый прототип, позволяющий полноценно сыграть как минимум одну партию.

### Фаза 4: Итеративная разработка и улучшение (годы)
**Цель**: доработка игры, улучшение пользовательского опыта.
Содержание работ:
1. Реализация минимального количества разнообразных игровых сценариев, реализация режима песочницы;
2. Добавление возможности генерации пользовательских карт на основе шума Перлина или графического редактора;
3. Создание редактора сценариев;
4. Тестирование с привлечением сообщества, сбор обратной связи, доработка возможных белых пятен и исправление возможных ошибок;
5. Разработка вики-подобной документации для авторов контента и добровольцев иного толка.
